    In order for the lab participants to complete the lab, the HCM instance in the lab needs to be set up ahead of time by someone with knowledge of the lab and environment.

    Set up the Digital Assistant

    As the instructor, you first need to go through all of the Set Up Your Digital Assistant and HCM Skill section of this lab yourself to set up a digital assistant and channel, which you then will associate with the HCM instance.

    Set up the Channel

    1. Click to open the side menu.
    2. Click Development and select Channels.
    3. Click again to collapse the side menu.
    4. On the Users tab, click + Channel.
    5. In the Create Channel dialog, fill in the Name, Description, and App Display Name fields.
    6. From the Channel Type dropdown, select Web.

      The completed dialog might look something like this:

    7. Click Create.

      Once created, you are returned to the page for the channel.

    8. Open the Route To dropdown menu, enter your DA name in the Filter box, and select your DA.
    9. Switch Channel Enabled to ON.
    10. Copy the the value of the App Id field to a text file.

    Enable the Digital Assistant in HCM Cloud

    Once you have set up the digital assistant and channel, you do the following to enable the digital assistant in HCM Cloud.

    Set the Necessary Profile Options

    Y enable your digital assistant in the HCM environment by setting the following two Profile Options:

    • FA_ENABLE_DIGITAL_ASSISTANT: You'll set this to Y to enable the digital assistant.
    • FA_DIGITAL_ASSISTANT_APP_ID: You'll set this to the app ID for your digital assistant.

    Here are the steps for adding the profile options:

    1. Log In to the HCM Cloud Instance as user hcm_impl
    2. Click on the arrow placed on right side of the profile icon.

      On clicking, a drop-down menu will appear.

    3. Select Setup and Maintenance.
    4. Click the Tasks icon on the right side of the screen.
    5. Click Search.
    6. Search for Manage Administrator Profile Values and select it from the list.
    7. Select Manage Administrator Profile Values.
    8. In the Profile Option Code field, enter FA_ENABLE_DIGITAL_ASSISTANT and click the Search button.
    9. In the Profile Value field for the Site profile level, enter Y.

      This will enable the digital assistant.

    10. In the Profile Option Code field enter FA_DIGITAL_ASSISTANT_APP_ID and click Search.
    11. In the Profile Value field for the Site profile level, enter the app ID for your digital assistant.

    Associate Privileges to Use the Digital Assistant

    You will need to give the user the following privileges so that the digital assistant can access the appropriate REST APIs.

    • Function Security Privileges
      • Use REST Service - Person Notes Visibility Options Read Only
      • Use REST Service - Candidates Read Only
      • Use REST Service - Job Applications Read Only
      • Use REST Service - Job Requisitions Read Only
    • Abstract Roles
      • Use REST Service - Benefit Enrollments Read Only
      • Use REST Service - Performance Goals Read Only
      • Use REST Service - Payslips Read Only
      • Use REST Service - Person Notes
      • Use REST Service - Performance Evaluations Read Only
      • Use REST Service - Public Workers Read Only
      • Use REST Service - Absences Read Only
      • Use REST Service - Job Offers Read Only

    Add Function Security Privileges

    1. Click Add Function Security Policy.
    2. In the Search dropdown, select Privileges.
    3. In the search field, enter Use REST Service - Person Notes Visibility and press Enter.
    4. Click Add Privilege to Role.
    5. Repeat above steps for each of these three security privileges:
      • Use REST Service - Candidates Read Only
      • Use REST Service - Job Applications Read Only
      • Use REST Service - Job Requisitions Read Only

      After completing these steps, you should the four security privileges under the Privileges

    Add Abstract Roles

    1. Click Next, skip step 4, and click again to move to the Role Hierarchy section.
    2. Click Add Role.

      The Add Role Membership window appears.

    3. In the Search field, enter Find Use REST Service - Benefit Enrollments Read Only and press Enter.
    4. Click Add Role Membership.
    5. Repeat the previous two steps for each of the following roles:
      • Use REST Service - Performance Goals Read Only
      • Use REST Service - Payslips Read Only
      • Use REST Service - Person Notes
      • Use REST Service - Performance Evaluations Read Only
      • Use REST Service - Public Workers Read Only
      • Use REST Service - Absences Read Only
      • Use REST Service - Job Offers Read Only

      After adding these roles, they should all be listed on the page.

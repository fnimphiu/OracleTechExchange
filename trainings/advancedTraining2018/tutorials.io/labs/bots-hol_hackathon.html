Oracle Intelligent Bots Advanced Training - Lab 11 (Hackathon)

Now that we've reached the end of the training, we want to give you the opportunity to put into practice what you've learned over the last couple of days.

You can start with the bot flow to track orders, which is not implemented in any of the versions of the bot that we have provided. So, building this flow track order flow is all up to you. We'll contribute the requirement specification and the help of the presenters in the room.

Prerequisites

  • A JavaScript IDE
  • Your clone of advt24hrsflowers_bot11.

Requirement Spec for the Track Order Flow

All orders that users enter to the system are saved in a storage collection in Oracle Anonymous Mobile Cloud Service. The 24hrsflowers bot has an order tracking function that allows users to see the current status of an order and to cancel it, even if the status is not in "delivered" or "active".

You probably have already noticed the presence of the Track Order Status item in the main menu:

In addition, the order tracking function should be accessible through natural language input like "I want to check the status of my order 24hrsflwrs-1532669587639".

All submitted orders should be stored in the 24hrsflowersOrders storage collection in AMCe. The order entries should be saved with the object ID being the order number. You need to be able to access an order and display relevant information to the user.

Here's an example of a JSON object that is returned from the storage collection: 

{
   "order": {
        "orderId": "24hrsflwrs-1532669587639",
        "orderStatus": "delivered",
        "productName": "Roses",
        "pricePerUnit": 5,
        "orderQuantity": 12,
        "discountPercent": 0
    },
    "payment": {
        "paymentDetails": "AMEX, 1234567"
    },
    "delivery": {
        "deliveryType": "overnight",
        "deliveryCost": 10
    },
    "recipient": {
        "recipientName": "Nic",
        "recipientAddress": "Home Address",
        "recipientMobile": "223344"
    },
    "card": {
        "cardSenderName": "Frank",
        "cardMessage": "Happy Valentine Day"
    }
}

The user should be able to change an existing order, which would mean being directed to human agent support. For the hackathon, it is enough to navigate the user to a state designed for handling human agent interaction. You don't need to implement the human agent interaction. The state can merely display a message.

Users should also be able to cancel an order if the status of the order isn't "delivered". "Active" orders that are cancelled should be removed from the Oracle Mobile Cloud 24hrsflowersOrders storage collection.

At the end of the track order flow, the bot should confirm the user action and ask whether users want to check the status of another order. When the users decline, they should be directed to the orderWhatIsNext state.

Implementation Hints

Obviously there is no way around building a custom component for this. To help get you started, your custom component service contains a custom component skeleton for this. See the custom component service's is/component/order/TrackOrder.js file.

You can change the content of the file as you need. It's yours. Also, it is your design decision what the custom component handles, how it handles it, and what is defined in OBotML in the dialog flow. For the dialog flow, the choice of bot system components and the use of Apache FreeMarker expressions and directives is completely yours.

Getting Started

If you don't already have a clone of advt24horsflowers_bot11, create one:

  • On the main Bots page in your instance, locate the tile for the advt24horsflowers_bot11, click , select Clone, and name the clone <YourUniqueInitials>_24horsflowers_bot11.

Good luck!
